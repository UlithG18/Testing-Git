Tailwind CSS – Cheat Sheet

Referencia práctica y explicada para maquetar rápido sin CSS puro

Layout (Estructura)

Tailwind permite crear layouts sin escribir CSS usando Flexbox y Grid. Estas clases definen cómo se organizan los elementos en la página.

Flexbox

  • flex: activa flexbox
  • flex-row / flex-col: dirección
  • justify-start | center | between
  • items-start | center | end
  • gap-1 a gap-12: espacio entre hijos

Grid

  • grid: activa grid
  • grid-cols-1 a grid-cols-12
  • col-span-1 a col-span-12
  • gap-1 a gap-12
  • sm:grid-cols-2, md:grid-cols-3

Tamaños

  • w-full / h-full
  • min-h-screen
  • max-w-sm a max-w-2xl
  • container mx-auto

Espaciado

El espaciado controla márgenes, relleno interno y separación entre elementos. Es clave para una interfaz limpia y legible.

  • Margin: m-1..12, mt-, mb-, ml-, mr-, mx-, my-
  • Padding: p-1..12, pt-, pb-, pl-, pr-, px-, py-
  • Separación entre hijos: space-x-2, space-y-4

Tipografía

Tailwind permite controlar texto sin CSS adicional: tamaño, peso, color, alineación y espaciado.

  • Tamaño: text-xs a text-5xl
  • Peso: font-thin a font-extrabold
  • Color: text-gray-500, text-blue-600, text-white
  • Alineación: text-left, center, right, justify
  • Tracking y line-height: tracking-wide, leading-relaxed

Colores y fondos

Los colores en Tailwind siguen una escala numérica. Esto ayuda a mantener consistencia visual.

  • Fondos: bg-gray-50, bg-white, bg-blue-500
  • Texto: text-gray-700, text-indigo-600
  • Hover: hover:bg-blue-700, hover:text-white
  • Focus: focus:ring-2 focus:ring-blue-400

Bordes y sombras

Usados para separar secciones y dar jerarquía visual.

  • border, border-2, border-t, border-b
  • border-gray-300, border-indigo-500
  • rounded, rounded-md, rounded-lg, rounded-full
  • shadow-sm a shadow-2xl

Efectos y transiciones

Permiten mejorar la experiencia del usuario sin JavaScript.

  • transition, transition-colors, transition-all
  • duration-150 a duration-500
  • hover:scale-105
  • opacity-0 a opacity-100

Responsive Design

Tailwind es mobile-first. Las clases se aplican desde el tamaño más pequeño.

  • sm (640px), md (768px), lg (1024px), xl (1280px)
  • Ejemplo: p-4 md:p-6 lg:p-8

Combos útiles

Combinaciones listas para usar en dashboards y formularios.

  • Botón: px-4 py-2 bg-blue-500 text-white rounded hover:bg-blue-700 transition
  • Card: p-6 bg-white rounded-lg shadow hover:shadow-xl
  • Grid responsive: grid grid-cols-1 md:grid-cols-3 gap-6
  • Input: p-2 border rounded focus:ring-2 focus:ring-blue-400 w-full

Buttons – Estados y variantes

Los botones suelen evaluarse por sus estados. Tailwind permite manejarlos sin JS.

  • Hover: hover:bg-indigo-700 hover:text-white
  • Active: active:scale-95
  • Focus: focus:outline-none focus:ring-2 focus:ring-indigo-500
  • Disabled: disabled:opacity-50 disabled:cursor-not-allowed
  • Ejemplo: px-4 py-2 bg-indigo-600 text-white rounded hover:bg-indigo-700 active:scale-95 disabled:opacity-50

Forms – Inputs avanzados

Formularios bien estilizados son clave en dashboards y perfiles.

  • Placeholder: placeholder-gray-400
  • Estados inválidos: border-red-500 text-red-600
  • Select: appearance-none bg-white
  • Textarea: resize-none
  • Input completo: w-full px-3 py-2 border rounded focus:ring-2 focus:ring-indigo-500

Position y Z-index

Esencial para modales, dropdowns y tooltips.

  • position: relative, absolute, fixed, sticky
  • Ubicación: top-0, right-0, inset-0
  • Z-index: z-10, z-20, z-50
  • Modal backdrop: fixed inset-0 bg-black/50 z-50

Overflow y Scroll

Muy usado en listas largas, tablas y modales.

  • overflow-hidden
  • overflow-auto
  • overflow-y-scroll
  • max-h-64, max-h-screen

Imágenes y media

Importante para perfiles, avatares y cards.

  • object-cover (imprescindible)
  • object-contain
  • rounded-full (avatares)
  • w-10 h-10, w-24 h-24
  • Avatar: w-12 h-12 rounded-full object-cover

Tablas

Frecuentes en paneles de administración.

  • table-auto, w-full
  • border-collapse
  • text-left text-sm
  • odd:bg-gray-50

State Variants

Permiten estilos condicionales sin JavaScript.

  • hover:
  • focus:
  • active:
  • disabled:
  • group-hover:
  • peer-focus:

Dark Mode

Cada vez más común en pruebas modernas.

  • dark:bg-gray-900
  • dark:text-white
  • dark:border-gray-700
  • Ejemplo: bg-white dark:bg-gray-800 text-black dark:text-white
